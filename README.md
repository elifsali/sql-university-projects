# Employee Management Database System (MS Access)

## 📌 Overview
Academic project focused on designing and normalizing a relational database for employee management in a telecommunications company.

The goal was to transform a non-normalized dataset into a structured relational database and implement SQL queries, forms, and reports.

---

## 🧠 Key Objectives
- Database normalization (1NF, 2NF, 3NF)
- Design of relational structure (Employees, Departments, Positions)
- SQL query development (JOIN, GROUP BY, aggregates)
- Data analysis and reporting in MS Access

---

## 🗄️ Database Design
- Employees
- Departments
- Positions

Relationships:
- Departments → multiple positions
- Positions → multiple employees
- Salary depends on position only

---

## 📊 SQL Queries
- Employees with salary = 900 BGN (JOIN)
- Number of employees per department
- Employees with salary ≤ 1000 BGN
- Employees whose name starts with “I”
- Average salary and total employee count
- Gross salary calculation

Gross Salary formula:
Base Salary + Base Salary × Work Experience × 0.01

---

## 📈 Reports & Forms
- Employee report grouped by position
- Sorted by name
- Summary: min, max, average salary
- Department form with employee subform
